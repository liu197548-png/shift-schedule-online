# 智能排班工具 - 在线版

一个功能完整的网页版智能排班系统，支持多班次、多组别、个人偏好和假期安排。

## 在线访问

部署后可通过以下链接访问：
`https://你的用户名.github.io/shift-schedule-online`

## 功能特性

### 1. 班次配置
- 自定义班次名称、时间段、颜色标识
- 设置每个班次的最低在班人数要求
- 支持添加/删除班次

### 2. 人员管理
- 创建多个组别
- 添加/删除人员
- 按组别管理员工

### 3. 特殊要求配置
- **休息日偏好**：为每个人员标记希望的休息日期
- **班次偏好**：设置人员对特定班次的偏好（多排/少排/正常）
- **假期设置**：标记节假日，可设置优先休息、必须上班或正常排班
- **固定班次**：设置特定人员在特定日期必须上的班次

### 4. 排班约束参数
- 班次均衡权重
- 连续同班次奖励
- 晚→早衔接惩罚（防止不合理的班次衔接）

### 5. 排班算法
- 自动根据在班人数分配班次
- 尊重个人偏好和固定班次设置
- 避免晚班后接早班等不合理衔接
- 确保班次均衡分布

### 6. 结果展示与导出
- 可视化排班表格，带颜色标识
- 显示每日在班人数和排班率统计
- 支持导出为 Excel 文件

## 部署到 GitHub Pages

### 方法一：自动部署（推荐）

1. **Fork 或创建仓库**
   - 在 GitHub 上创建一个新仓库，命名为 `shift-schedule-online`
   - 将本项目的所有文件上传到该仓库

2. **启用 GitHub Pages**
   - 进入仓库的 **Settings** > **Pages**
   - 在 **Source** 部分选择 **GitHub Actions**
   - 系统会自动使用 `.github/workflows/deploy.yml` 进行部署

3. **访问网站**
   - 部署完成后，访问 `https://你的用户名.github.io/shift-schedule-online`

### 方法二：手动部署

1. 进入仓库的 **Settings** > **Pages**
2. 在 **Source** 部分选择 **Deploy from a branch**
3. 选择 **main** 分支和 **/(root)** 目录
4. 点击 **Save**

## 使用说明

1. **配置班次**：在"班次配置"部分设置你的班次信息
2. **添加人员**：在"人员管理"部分创建组别并添加人员
3. **设置特殊要求**：
   - 设置休息日偏好
   - 配置班次偏好
   - 标记假期
   - 设置固定班次（如有需要）
4. **调整约束参数**（可选）：根据需要调整排班算法参数
5. **生成排班表**：点击"生成排班表"按钮
6. **导出结果**：点击"导出Excel"下载排班表

## 配置保存与加载

- **保存配置**：点击"保存配置"按钮，将当前配置下载为 JSON 文件
- **加载配置**：点击"加载配置"按钮，选择之前保存的 JSON 文件

## 技术说明

- 纯前端实现，无需后端服务器
- 使用原生 JavaScript，无需构建工具
- 数据存储在浏览器本地，刷新页面不会丢失
- 支持配置导出/导入，便于备份和分享
